### Why These Technologies?

| Layer | Technology | Rationale |
|-------|------------|-----------|
| **Monorepo** | Turborepo | Efficient build orchestration, remote caching, dependency graph optimization |
| **Linting** | BiomeJS | 10x faster than ESLint+Prettier, unified tool, simpler config |

| **API** | Fastify | 2x faster than Express, built-in validation, excellent TypeScript support |
| **Auth** | Firebase Auth | Built-in social providers, secure token management, free tier |
| **Database** | Supabase | Open source Firebase alternative, PostgreSQL, real-time subscriptions |
| **State** | Zustand | Minimal boilerplate, excellent TypeScript, no provider needed |
| **Validation** | Zod | Runtime type checking, great DX, easy error messages |

## Architecture Decisions

### ADR 001: Feature-Based Folder Structure

**Decision:** Organize code by feature rather than by type.

```
âœ… Feature-based:
src/features/transactions/
  â”œâ”€â”€ api/
  â”œâ”€â”€ screens/
  â”œâ”€â”€ stores/
  â””â”€â”€ index.ts

âŒ Type-based:
src/screens/transactions/
src/stores/transactions/
src/api/transactions/
```

**Rationale:**
- Cohesion: Related code lives together
- Scalability: Easy to add/remove features
- Discoverability: Find everything in one place

### ADR 002: Zustand for State Management

**Decision:** Use Zustand instead of Redux or Context API.

**Rationale:**
- Minimal boilerplate
- Excellent TypeScript inference
- No providers needed
- Built-in persistence middleware
- Smaller bundle size

### ADR 003: Fastify over Express

**Decision:** Use Fastify as the API framework.

**Rationale:**
- 2x faster request handling
- Built-in schema validation
- Native async/await support
- Better plugin architecture
- Automatic OpenAPI generation

### ADR 004: Supabase + Firebase Hybrid

**Decision:** Use Firebase for Auth and FCM, Supabase for database.

**Rationale:**
- Firebase Auth: Best-in-class social auth
- Firebase FCM: Reliable push notifications
- Supabase: Full PostgreSQL power
- Supabase: Row Level Security
- Cost: Both have generous free tiers

### ADR 005: TypeScript Strict Mode

**Decision:** Enable strict TypeScript across all packages.

**Rationale:**
- Catch errors at compile time
- Better IDE support
- Self-documenting code
- Easier refactoring
